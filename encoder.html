Cipher Info

  • Base64: Encodes text into a 64-character alphabet. Not for secrecy — transport-friendly.
  • Base32: Encodes bytes into a 32-character alphabet (RFC4648). Useful for case-insensitive tokens.
  • Hex: Encodes bytes to hexadecimal pairs. Common for binary/text interchange.
  • Caesar: Shift letters by a numeric shift (1-25). Enter number in the key input.
  • ROT13: Simple letter rotation (13). Symmetric and used for simple obfuscation.
  • Morse Code: Dots (.) and dashes (-). Use space between letters and / for word separators.
  • URL Encode/Decode: Percent-encoding for transport in URLs.
  • XOR: Symmetric XOR with a repeating key. Binary-safe; output is Base64.
  • AES-256: Encrypt/Decrypt using passphrase (CryptoJS). For PBKDF2-derived keys use AES-256 (PBKDF2).
  • AES-256 (PBKDF2): Derive encryption key using PBKDF2 with embedded salt for stronger keying.
  • Vigenère: Polyalphabetic substitution using an alphabetic keyword.
  • SHA-256: Cryptographic hash (one-way). Use under Encrypt to compute the digest.
  • MD5: Legacy hash function (one-way). Not recommended for security use-cases.
